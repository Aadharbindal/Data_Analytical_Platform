6. Processing Workflow
Upload → Validate → Virus/File Checks → Store Raw File → Register Metadata → Detect Schema → Profile Dataset → Persist Profile → Ready for Analytics
7. Validation Rules
Accept CSV and XLSX only.
Reject empty files.
Reject unsupported encodings where possible.
Detect duplicate column names.
Validate maximum file size.
Report malformed rows with line numbers.
8. AntiGravity Prompt
Generate a production-grade Dataset Engine in FastAPI. Implement modular upload APIs, asynchronous file handling, CSV/XLSX parsing, schema inference, profiling, metadata extraction, SQLAlchemy models, Alembic migrations, object storage abstraction, validation rules, structured logging, unit tests and clean architecture. Expose REST endpoints for upload, metadata, profile and deletion. Keep the implementation extensible for future cloud storage and streaming ingestion.
